# ADR-009: 로컬 전환 PoC — 임베딩은 대체 가능, 판정 LLM은 CPU 서빙 불가

- **Status**: Accepted
- **Date**: 2026-07-15

## 배경 (Context)

W09 설계 문서의 Tier 2(완전 on-premise)가 실현 가능한지, §7 전환 검증 절차를
실제 로컬 구성(WSL2, CPU only, RAM 7.3GB, Ollama)에서 실행했다. 후보:
임베딩 bge-m3, 판정 LLM qwen2.5:7b·exaone3.5:7.8b. 프롬프트(judge_v2.1)와
평가 기준은 동결 — 변수는 서빙 환경뿐.

## 대안 검토 (Alternatives Considered)

PoC 중단 시점의 선택지:

- **"CPU 서빙 불가"로 결론·문서화 (채택)**: OOM 자체가 Tier 2 질문에 대한
  결정적 답.
- **컨텍스트 축소(3072) 재시도**: 판정 입력이 원칙당 ~6천 토큰이라 입력이
  잘림 — judge_v2.1이 아닌 다른 것을 측정하게 되어 방법론적으로 무효.
- **exaone 확인 사살**: qwen(4.7GB, 디스크 4.4GiB)보다 큰 모델(4.8GB)이라
  동일 하드웨어 OOM은 실측 없이 확정 — 시스템 위험만 추가.

## 결정 (Decision)

**PoC를 다음 실측 결론으로 종료한다:**

1. **임베딩은 로컬 대체 가능** — bge-m3 recall@5 = 0.90 (게이트 ≥0.8 통과,
   기준 3-large 1.00). 유일한 MISS는 제21조(기지의 최약 원칙).
2. **판정 LLM의 CPU 서빙은 불가** — qwen2.5:7b가 원칙당 2.5~4.5분(gpt-4.1
   대비 ~85배)이며, 광고 규제(구성요건 F1~F11)의 대형 JSON 출력을 닫지 못하고
   폭주 → KV 메모리 잠식 → 스왑 소진 → OOM. 워치독이 시스템 보호를 위해
   강제 종료. 단지 느린 것이 아니라 **메모리 불안정**이다.
3. 따라서 **Tier 2의 실질 전제는 GPU 인프라**이고, 진입 시 judge_v2.1 출력
   스키마 경량화(checks maxItems, quote 축약)를 함께 검토한다 — 이는 §7
   품질 게이트 재검증을 다시 요구한다.

## 영향 (Consequences)

- ✅ 설계 문서 §3의 하이브리드 권장이 실측으로 보강됨
- ✅ Tier 1.5 변형 성립: 임베딩까지 로컬(bge-m3)로 내려 반출면을 "판정 호출"
  하나로 좁힐 수 있음 — 대가는 recall 1.00→0.90 + 전체 재인덱싱
- ⚠️ 판정 LLM의 로컬 품질(정확도/오탐)은 미측정으로 남음 — GPU 환경 확보
  시 §7 게이트로 측정하는 것이 후속 작업
- ⚠️ 출력 스키마 경량화는 스키마 변경(명세서 §7.3)이므로 착수 시 별도 ADR
