# ADR-007: FR-09 재설계 — confidence 임계값 폐기, 인용 중복 탐지 + 판정 캐시

- **Status**: Accepted
- **Date**: 2026-07-14

## 배경 (Context)

명세서 FR-09는 "판정 신뢰도 임계값 미달 시 보류"로 정의됐으나, W08 실측에서
모델의 self-reported confidence는 변별력이 없음이 확인됐다 — 정답 위반과
오탐의 confidence가 최소값부터 겹치고(둘 다 0.95), 임계값을 0.60~0.95 어디에
둬도 잔존 오탐이 걸러지지 않으며, 0.95 초과로 올리면 진짜 위반까지 죽는다.
0.50대 값은 이미 NEEDS_REVIEW인 셀에만 나타난다.

별건으로 NFR-07(재현성)도 미충족 상태다: temperature=0 + seed 고정에도 동일
입력 2회 실행에서 186셀 중 6셀(3.2%)이 뒤집힌다. 감사 대응 도구는 "같은
문구를 같게 판정한다"를 보장해야 한다.

## 대안 검토 (Alternatives Considered)

- **confidence 임계값 (명세서 원안)**: 실측으로 무효 확인 — 위 배경.
- **인용 문구 중복 탐지 (채택)**: 잔여 오탐의 정체가 "한 행위를 이웃 원칙에
  돌려쓰기"임에 착안, 같은 입력 문구가 복수 원칙의 구성요건 근거로 재사용되면
  강등. 평가셋에 맞춘 휴리스틱이 아니라 라벨링 규칙 2("하나의 행위는 하나의
  조문")의 코드화. 법정 중복쌍(제21조①의 단정적 표현 ↔ 제19조③의 왜곡 —
  문언이 동일)만 예외 허용. 비용 0.
- **자기일관성 N회 실행**: 비용 N배 대비 뒤집힘 1.4~3.2%로 효과 제한적 — 기각.

## 결정 (Decision)

1. **FR-09의 오탐 억제 신호를 confidence 임계값에서 인용 문구 중복 탐지로
   교체한다** (법정 중복쌍 예외). confidence 필드는 스키마에 유지하되 게이트
   신호로 쓰지 않는다.
2. **판정 캐시로 재현성을 보장한다**: 동일 (input_hash, prompt_version,
   model) 재요청 시 점검 이력(memory)의 저장된 판정을 반환. 신규 판정은
   명시적 재판정 요청 시에만. LLM의 비결정성은 남지만 도구의 출력은
   결정적이 된다 (NFR-07의 충족 지점을 "모델"에서 "도구"로 이동).

명세서 FR-09/NFR-07 문구 갱신 필요 (본 ADR 인용).

## 영향 (Consequences)

- ✅ 오탐 게이트가 조문 근거 규칙로 동작 — 모델 자기평가에 의존하지 않음
- ✅ 감사 관점에서 동일 입력 → 동일 출력 보장, 재판정은 이력에 명시적 기록
- ⚠️ 중복 탐지 규칙은 잔여 오탐 관찰에서 발견됨 — 두 평가셋에서 진짜 위반
  (특히 h-07의 복수 구성요건, 법정 중복쌍 문항)을 죽이지 않는지 검증 필수
- ⚠️ 판정 캐시는 prompt_version/model 변경 시 자동 무효화 — 키에 포함할 것
- ⚠️ 오탐 프로필 기록: 순수 정상 문항 오탐 0건, 잔여 오탐은 위반 문항의 이웃
  원칙 추가 지적에 국한 — 스크리닝 도구로 수용 가능한 프로필

## 참고 (Notes)

confidence 무변별은 gpt-4.1 + judge_v2.1 기준 실측이다. 모델/프롬프트가
바뀌면 재검토 여지가 있으나, 자기평가 신뢰는 기본적으로 지양한다.
