# ADR-002: 6대 원칙 태그를 금소법 본법 제17~22조로 한정

- **Status**: Accepted
- **Date**: 2026-07-13

## 배경 (Context)

청크 메타데이터의 원칙 태그는 principle_filter 검색의 기준이다. 금소법 본법은
제17~22조가 6대 원칙과 1:1 대응하지만, 시행령·감독규정은 같은 조 번호라도
내용이 달라 번호 기반 매핑이 불가능하다. 태그를 어디까지 부여할지 결정 필요.

## 대안 검토 (Alternatives Considered)

- **본법 제17~22조만 태그**: 매핑 근거가 명확해 태그 신뢰도 100% — 필터 사용 시
  시행령의 세부 기준(예: 광고 필수 포함사항)이 검색 대상에서 빠짐.
- **시행령·감독규정까지 수동 조문 대응표 작성**: 실무적으로 가장 정확 — 조문
  대응표 구축·유지 비용이 크고 W06 범위 초과.
- **LLM으로 자동 태그**: 비용 낮음 — 태그 오류가 필터 검색 누락으로 조용히
  전파되고, 환각 통제라는 프로젝트 원칙과 상충.

## 결정 (Decision)

**W06은 본법 제17~22조만 태그.** 잘못된 태그는 없는 태그보다 위험하다는 판단.
principle_filter는 보조 수단이고 기본 검색은 전체 코퍼스 대상이라, 시행령
조항도 유사도 검색으로는 여전히 잡힌다.

## 영향 (Consequences)

- ✅ 태그된 청크는 매핑 근거가 법조문 수준으로 명확
- ✅ 유사도 검색 경로로 시행령·감독규정 접근성 유지
- ⚠️ principle_filter 단독 사용 시 시행령 세부 기준 누락 — W07 판정 엔진은
  "본법 태그 필터 + 시행령 보강 검색" 조합으로 설계해야 함
- ⚠️ 후속 작업: W07 시작 시 이 항목 정산 (필터 확장 or 조문 대응표 도입 재검토)

## 참고 (Notes)

W07 판정 엔진 설계 시 이 ADR의 후속 작업을 먼저 논의. 조문 대응표를 도입하게
되면 별도 ADR로 기록하고 본 ADR을 Superseded 처리.
