# ADR-003: 임베딩 모델을 text-embedding-3-large로 변경

- **Status**: Accepted
- **Date**: 2026-07-13

## 배경 (Context)

명세서 §7.1은 비용/성능 균형을 이유로 text-embedding-3-small을 지정했다.
그러나 W06 DoD 검증에서 recall@5 = 0.20으로 검색이 사실상 작동하지 않았다
(기대 조항이 21~162위). 한국어 법조문은 "금융상품판매업자등은 … 대통령령으로
정하는 …" 같은 상용구 비중이 높아, 3-small의 표현 해상도로는 조문 간
구별이 되지 않는다.

## 대안 검토 (Alternatives Considered)

변형 6종 실측 (질의 10건, recall@5):

| 조건 | recall@5 |
|---|---|
| 3-small (명세서 §7.1) | 0.20 |
| 3-small + 호 단위 세분화 | 0.20 |
| 3-small + 어휘 하이브리드 | 0.20 |
| **3-large (채택)** | **0.70** |
| 3-large + 호 세분화 | 0.40 |
| 3-large + HyDE | 0.80 |

청크 세분화와 어휘 하이브리드는 직관과 달리 무효 또는 유해했다. HyDE는
+0.10이지만 질의마다 LLM 호출이 추가돼 지연·비용이 검색 단계에 상시 발생.

## 결정 (Decision)

**text-embedding-3-large 단독 채택.** 모델 변경만으로 0.20 → 0.70이며, 평가
라벨 정정(ADR-004) 후 1.00. 비용은 6.5배지만 390청크 전체가 몇 센트 수준이고
캐싱으로 재과금 없음. HyDE는 현 스코어에서 불필요한 복잡도로 판단, 보류.

## 영향 (Consequences)

- ✅ W06 DoD 통과 (recall@5 = 1.00, 목표 0.80)
- ✅ 명세서 §7.1/§12 갱신 필요 — 갱신 시 본 ADR을 사유로 인용
- ⚠️ 임베딩 차원 증가로 인덱스 크기·검색 메모리 증가 (390청크 규모에선 무시 가능)
- ⚠️ W09 on-premise 설계 시 로컬 임베딩 모델 대체 검토에 같은 실측 절차 필요
  (한국어 법조문 구별력이 채택 기준)

## 참고 (Notes)

HyDE 재검토 조건: 코퍼스 확장(타 법령 추가)으로 recall이 목표 아래로
떨어질 때. 그 경우 별도 ADR로.
