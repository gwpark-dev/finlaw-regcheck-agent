# ADR-001: 법령 PDF 파싱 전략 (부칙 제외·줄바꿈 복원)

- **Status**: Accepted
- **Date**: 2026-07-13

## 배경 (Context)

W06 지식베이스 구축은 국가법령정보센터 PDF 5종을 조(條) 단위로 청킹하는 것이
전제다. 구현 중 세 가지 함정 발견: (1) 페이지마다 "법제처 N 국가법령정보센터"
머리말·꼬리말 반복, (2) 부칙은 조 번호가 제1조부터 다시 시작해 본문 조항과 충돌,
(3) 한글 PDF는 단어 중간 줄바꿈("금융\n소비자")으로 원문이 훼손됨. 조항 원문
인용이 판정 근거가 되는 시스템이라(NFR-02) 원문 복원 정확도가 곧 신뢰성이다.

## 대안 검토 (Alternatives Considered)

- **PDF + 전처리 규칙**: 수동 다운로드만으로 동작, 외부 의존 없음 — 전처리
  규칙이 법제처 PDF 레이아웃에 결합됨.
- **법령정보 Open API (XML)**: 조 단위 구조화 데이터라 파싱 함정 자체가 없음 —
  API 키 신청·승인 대기 필요, W06 데모 범위 초과.
- **HWP/텍스트 수동 복사**: 도구 불필요 — 재현 불가능, 법령 개정 시 갱신 절차가
  사람 손에 전적으로 의존.

## 결정 (Decision)

**PDF + 전처리 규칙 3종 선택.** pdfplumber 추출 후 머리말·꼬리말 제거, 부칙
제외, 구조 기호(제N조·①②③ 등)로 시작하지 않는 줄은 앞줄에 이어붙이는 줄바꿈
복원을 적용. Open API의 이점은 인정하나 키 발급 대기가 W06 일정과 맞지 않고,
전처리 규칙은 이미 검증됨(법령 5개 → 390청크).

## 영향 (Consequences)

- ✅ 수동 다운로드만으로 파이프라인 전체 재현 가능
- ✅ 조 번호 충돌 없는 깨끗한 청크 확보
- ⚠️ 부칙 제외로 시행일·경과조치 정보 손실 — 판정 근거로 부칙이 필요해지면 재검토
- ⚠️ 줄바꿈 복원 오류는 원문 인용 훼손으로 직결 — W06 DoD 육안 검증에 원문
  대조 포함
- ⚠️ 법제처 PDF 레이아웃 변경 시 전처리 규칙 깨짐

## 참고 (Notes)

법령 개정 시 재수집·재인덱싱 절차는 W09 아키텍처 설계에서 다룸. Open API 전환을
결정하게 되면 별도 ADR로 기록.
