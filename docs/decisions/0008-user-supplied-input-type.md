# ADR-008: input_type은 사용자 필수 입력, classifier는 보조로 강등

- **Status**: Accepted
- **Date**: 2026-07-14

## 배경 (Context)

judge v2.1의 국면 게이트(제22조는 광고 국면만 적용)는 input_type에 의존한다.
W07~08 실험은 전부 oracle(정답 유형 주입)로 측정했으나 실서비스엔 oracle이
없다. 실측 결과 규칙 기반 classifier의 광고/상담 이진 정확도는 0.535로
"무조건 상담"(0.837)보다 나쁘고, 오분류 20건 전부가 상담→광고 방향이라
게이트가 열리는 쪽으로 역작동한다. 실험 성능(0.903/0.096)이 실서비스에서
재현되지 않는 구조적 격차다.

## 대안 검토 (Alternatives Considered)

- **사용자 필수 입력 + classifier 보조 (채택)**: 검수 담당자는 자기가 광고를
  검수하는지 상담 스크립트를 검수하는지 항상 안다(명세서 §3.1 — 사내 도구).
  classifier는 기본값 제안과 불일치 경고만 담당. 게이트가 정답 유형 위에서
  동작해 실험 성능이 실서비스에서 재현됨.
- **필수 입력 + classifier 제거**: 단순하나 상품군 분류(0.814)는 보강 검색
  질의에 유용해 버리기 아까움.
- **classifier 개선 투자 (W05 ML 이식 등)**: oracle 실험이 보여준 기대 상한이
  +0.021로 미미하고, 이진 분류가 개선돼도 오분류 시 게이트 역작동 위험 자체는
  남는다 — 기각.

## 결정 (Decision)

**input_type(광고/상담)을 사용자 필수 입력으로 받는다.** 유형은 판정
결과가 아니라 입력 사실이므로 사람이 지정하는 것이 자연스럽다. classifier는
(1) UI 기본값 제안, (2) 사용자 선택과 불일치 시 경고 표시, (3) 상품군 분류
(보강 검색 질의용)로 역할을 한정한다. 파이프라인은 `check(text, input_type=...)`
필수 인자로 강제하고, W10 Streamlit UI에 유형 선택을 필수 항목으로 넣는다.

명세서 FR-03 역할 조정 + UC-1 흐름 갱신 필요 (본 ADR 인용).

## 영향 (Consequences)

- ✅ 국면 게이트의 실서비스 신뢰성 확보 — oracle 실험치가 곧 실서비스 기대치
- ✅ classifier 개선에 시간 투자 불필요 (W05 ML 분류기 이식 계획 폐기)
- ⚠️ UI/API 사용자가 유형을 잘못 지정하면 게이트는 여전히 역작동 — 불일치
  경고가 완충. 사람의 오지정 확률은 classifier 오분류율(0.465)보다 훨씬 낮다고
  가정하며, 이 가정은 W10 시연에서 관찰
- ⚠️ 배치/API 경로는 호출자가 유형을 반드시 지정해야 함 (문서화 필요)

## 참고 (Notes)

W05 하이브리드 분류기 이식 계획(W06 프롬프트의 선택 항목)은 본 ADR로 공식
폐기. classifier의 이진 분류 결함 원인(AD_KEYWORDS의 범용 토큰, 광고 우선
tie-break)은 기본값 제안 품질을 위해 수정할 가치는 있으나 우선순위 낮음.
