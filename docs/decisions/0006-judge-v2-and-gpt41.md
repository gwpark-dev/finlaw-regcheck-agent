# ADR-006: 판정 엔진을 v2.1(2단 구성요건 판정)로, 판정 모델을 gpt-4.1로 변경

- **Status**: Accepted
- **Date**: 2026-07-14

## 배경 (Context)

W07 측정에서 판정 정확도 0.565 / 오탐률 0.482로 목표(0.75 / 0.15)에 크게
미달했다. 오탐의 양상은 "문제가 하나라도 보이면 6대 원칙에 폭넓게 VIOLATION"
이며, 라벨 주관성·검색 실패·스키마는 실측으로 원인에서 배제됐다. 명세서 §12의
모델(gpt-4o-mini) 변경은 ADR이 필요하므로, 구조와 모델의 기여를 분리하는
실험(A~H, 31건/186셀 동결 + 홀드아웃 12건)을 수행했다.

## 대안 검토 (Alternatives Considered)

기여도 분해 (실측, data/eval/w08_experiment_results.md):

| 변경 | 정확도 | 오탐률 |
|---|---|---|
| judge 구조 (v1.0 → v2.1) | +0.232 | −0.271 |
| 모델 (gpt-4o-mini → gpt-4.1) | +0.080 | −0.091 |
| classifier (규칙 기반 → oracle) | +0.021 | −0.018 |

- **구조 v2.1**: 판정을 [1단: 조문 구성요건별 충족/불충족 + 문구 인용 →
  2단: 코드 결정] 으로 분해하고, 조문의 결정적 적용 요건(제22조는 광고 국면만,
  제21조제6호는 투자성 상품만)을 프롬프트가 아닌 코드 게이트로 강제.
- **모델**: gpt-4o-mini(F)는 v2.1 위에서도 오탐률 0.187/0.152(홀드아웃)로
  목표 미달 + 독해 실패(명시 문장을 누락 판정) 지속 + 홀드아웃 미탐 1건.
  gpt-4.1-mini(G)는 0.175로 미달에 처리도 가장 느림(24.6초). gpt-4.1(H)만
  0.096/0.076으로 양 목표 통과. gpt-4o는 gpt-4.1 대비 입력단가 25% 비싸고
  지시 준수·긴 컨텍스트에서 열위로 판단해 실험에서 제외.

## 결정 (Decision)

**judge v2.1 구조 + gpt-4.1을 판정 모델로 채택.** 정확도 목표는 구조만으로
달성되지만(F 0.823), 오탐률 목표(≤0.15)는 gpt-4.1이 있어야 안정적으로
통과한다(0.096, 홀드아웃 0.076). 오탐은 이 도구의 핵심 리스크(명세서 §13
"오탐 과다 → 도구 불신")이므로 모델 상향 비용을 정당화한다.

과적합 검증: 신규 상품군·구성요건으로 구성한 홀드아웃 12건에서 H가 목표
유지(0.917/0.076), 순수 정상 문항 오탐 0건.

## 영향 (Consequences)

- ✅ 명세서 §7.2/§12 모델 갱신 필요 (gpt-4o-mini → gpt-4.1, 본 ADR 인용)
- ⚠️ 판정 비용 건당 약 $0.09 (mini 대비 ~15배). 내부 검수 도구 빈도 기준
  수용 가능하나 W09 on-premise 설계에서 비용·대체 모델 축으로 재검토
- ⚠️ 저비용 대안 기록: F(mini+v2.1) = 0.823/0.187 — 비용 제약 시나리오의
  fallback으로 유효
- ⚠️ 미결 ① 재현성: H도 동일 실행 2회에서 3.2% 뒤집힘 (NFR-07 미충족) → W08
- ⚠️ 미결 ② 국면 게이트가 input_type에 의존 — 실서비스에서 classifier
  오분류 시 게이트 역작동 (oracle 실험과 실서비스의 격차) → W08
- ⚠️ 공통 잔여 약점: 부당권유 원칙에서 포괄 조항식 구성요건 오지정

## 참고 (Notes)

실험 상세·원칙별 수치는 data/eval/w08_experiment_results.md. w07-030 라벨은
조문(시행령 제18조②)이 모델 해석을 뒷받침하지 않아 OK로 유지 — 채택 모델에
맞춘 라벨 변경을 하지 않았다는 기록을 남긴다. ADR-005의 판정 구조(본법 고정
주입 + 원칙별 개별 호출)는 유지되며 본 ADR은 그 위의 판정 방식 변경이다.
