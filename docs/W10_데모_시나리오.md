# W10 Streamlit 데모 시연 대본

> 명세서 §10 W10 DoD — 문구 입력 → 위반 리포트 E2E + 시연 시나리오 3종(위반/정상/보류).
> 실제 판정 결과는 `data/demo/scenarios.jsonl`을 `check()`로 사전 실행해 확인·동결.

## 실행

```bash
uv run streamlit run app.py --server.address localhost   # http://localhost:8501
```

내부망 localhost 전용(W09 §5 경계 원칙) — `--server.address localhost`로 127.0.0.1에만
바인딩(기본값은 전 인터페이스라 LAN 노출됨). 외부 공개 터널(cloudflared 등) 금지. 판정에는 OpenAI API가
쓰이므로 `.env`의 `OPENAI_API_KEY` 필요. 시나리오 3종은 사전 판정돼 **캐시에 있으므로**
검사 시 즉시 결과가 뜬다(판정 캐시 = 결정성, NFR-07).

## 시연 순서 (약 5분)

### 0. 화면 소개 (30초)
- 상단: 문구 입력 + **유형(광고/상담) 필수 선택**. 분류기가 유형을 제안하되 최종 선택은
  사람 몫이다(ADR-008). 다른 값을 고르면 경고 배지가 뜬다 — 시연해 보인다.
- 탭: `🔍 점검` / `📜 점검 이력`(UC-4).
- **결과 화면**: 검사하면 원칙 카드가 심각도 순 3개 섹션으로 묶여 나온다 —
  **🔴 위반 소지 → 🟡 검토 필요**(항상 펼침) → **🟢 문제 없음**(개수만 헤더에 표시,
  접혀 있음). 검수자는 문제 있는 것부터 본다. 카드가 없는 섹션은 아예 숨는다.
  기술 정보(판정 모델·기준 버전·문구 식별번호)는 **"판정 정보(감사 대응용)" expander**
  안에 쉬운 설명과 함께 접혀 있다.

### 1. 위반 시나리오 (60초) — 🔴 VIOLATION
- 유형 **광고** 선택 → 아래 문구 붙여넣기 → **검사**.

  > [△△증권 그로스펀드 광고] 연 8% 수익 보장! 원금은 저희가 책임지고, 혹시 손실이 나면
  > 전액 보전해 드립니다. 위험은 전혀 없습니다. 지금 가입하세요.

- **결과: 광고 규제(제22조) VIOLATION** — 근거 조항(제22조)·인용·수정안까지 표시.
  구성요건 **F1**(계약 전 설명서·약관 열람 권유 없음)·**F3**(투자 위험·과거 수익률
  비보장 고지 없음)이 함께 충족.
- 시연 포인트: **근거 조항 없는 판정은 없다**(NFR-02). 카드마다 법령·조항·원문 인용.
- 참고: 광고 문구의 위반은 제22조가 포섭하므로 "위반 2건"은 별개 원칙이 아니라 **구성요건
  2개(F1·F3)**로 나타난다. 꺾기 낀 대출광고로 불공정영업(제20조)까지 띄우는 변형도 가능하나,
  그 경우 게이트가 불공정영업을 NEEDS_REVIEW로 강등해 위반/보류 대비가 흐려져 이 문구로 확정.

### 2. 정상 시나리오 (45초) — 🟢 OK (오탐 억제)
- 유형 **상담** 선택 → 붙여넣기 → **검사**.

  > 고객님께서 먼저 문의 주신 펀드 상담입니다. 투자성향과 재산 상황, 투자 목적을 먼저
  > 확인하겠습니다. 이 상품은 원금 손실이 발생할 수 있고, 과거 수익률이 미래 수익을
  > 보장하지 않습니다. 투자설명서와 약관을 함께 보시고, 총 보수는 연 1.3% 수준입니다.
  > 충분히 검토하신 뒤 결정하시면 됩니다.

- **결과: 전 6원칙 OK.** 위험 고지·설명서 안내·고객 요청 상담이 갖춰져 오탐이 나지 않는다.
- 시연 포인트: **오탐 억제**(오탐률 목표 ≤0.15, W08 실측 0.018). 나쁜 문구만 잡고 정상은 통과.

### 3. 보류 시나리오 (60초) — 🟡 NEEDS_REVIEW (인용 중복 게이트)
- 유형 **상담** 선택 → 붙여넣기 → **검사**.

  > 고객님, 이 펀드 가입하실 때 투자성향이나 소득은 굳이 확인 안 하셔도 됩니다. 절차는
  > 제가 알아서 다 처리할 테니 그냥 믿고 맡기시면 됩니다.

- **결과: 적합성·적정성·설명의무·부당권유 4원칙이 NEEDS_REVIEW**(게이트 강등), 불공정영업·
  광고 규제는 OK.
- 시연 포인트: **하나의 행위(투자성향·소득 확인 생략)를 여러 원칙이 함께 지목**했다. 도구가
  귀속을 확정하지 못하면 `VIOLATION`을 `OK`로 내리지 않고 **NEEDS_REVIEW로 올려 사람에게
  보낸다**(UC-5). 각 카드에 "👤 사람 검토 필요" 안내. **위반을 정상으로 통과시키는 일은
  없다**(위반 누락률 0%, ADR-007 / CLAUDE.md 한계 2).

### 4. 판정 캐시 시연 (45초) — ⚡ 재현성 (NFR-07)
- 위반 문구(시나리오 1)를 **다시 검사** → 상단에 **⚡ 캐시된 판정** 배지 + 소요 시간이
  수십 ms로 표시된다. 같은 (문구, 프롬프트 버전, 모델)이면 저장된 판정을 그대로 반환.
- 시연 포인트: LLM은 temperature=0에도 재현이 안 되므로(실측 1.4~3.2% 뒤집힘), **도구
  출력을 캐시로 결정적**으로 만든다(ADR-007). 라이브 재판정을 보이려면 **♻️ 재판정
  (force_recheck)** 버튼 → 다시 6원칙을 판정(~12초)하고 **그 사실이 감사 로그에 남는다**.

### 5. 점검 이력 + 감사 로그 (60초) — 📜 UC-4 / NFR-05
- `📜 점검 이력` 탭 → 방금 점검한 3건이 최신순으로. 원문이 아니라 **입력 해시 + 요약**만
  (NFR-04: 원문 개인정보는 이력에 안 남긴다).
- 터미널에서 감사 로그를 열어 보인다:

  ```bash
  tail -n 3 data/audit/audit.jsonl | python3 -m json.tool
  ```

  각 레코드에 timestamp·input_hash·마스킹된 입력·검색 조항 ID·원칙별 verdict·근거 인용·
  게이트 강등 내역·`cache`(hit/forced_recheck)·meta(model/prompt_version/seed/
  system_fingerprint)가 append-only로 기록된다(FR-10, NFR-05/07).

## 사전 검증 결과 (동결)

`data/demo/scenarios.jsonl`을 `check(force_recheck=True)`로 실행한 실측(모두 NFR-01 30초 내):

| 시나리오 | 유형 | 판정 결과 | 화면 섹션 구성 | 소요 |
|---|---|---|---|---|
| demo-violation | 광고 | 광고 규제 **VIOLATION**(F1·F3), 나머지 5 OK | 🔴 위반 소지 1(광고 규제) · 🟢 문제 없음 5(접힘) | ~12초 |
| demo-clean | 상담 | 전 6원칙 **OK** | 🟢 문제 없음 6(접힘)만 — 위반·검토 섹션 숨김 | ~14초 |
| demo-review | 상담 | 적합성·적정성·설명의무·부당권유 **NEEDS_REVIEW**(게이트 강등), 2 OK | 🟡 검토 필요 4 · 🟢 문제 없음 2(접힘) — 위반 섹션 숨김 | ~12초 |

문구는 평가셋/홀드아웃과 겹치지 않는 새 입력(미학습)이다. 시연 중 fresh 재판정 시
LLM 비결정성으로 소폭 달라질 수 있으나, 캐시가 걸린 첫 검사는 위 결과로 결정적이다.
